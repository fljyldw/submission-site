# GPS 轨迹作业提交系统 - 项目结构说明

> 本文档帮助你快速了解项目的文件结构和功能，方便后续修改和维护。

---

## 一、项目概览

这是一个 **GPS 轨迹作业提交与自动审核系统**，学生上传 GPS 轨迹 CSV 文件，系统自动分析连续录制时长（要求 ≥5 小时），即时返回审核结果。管理后台可以查看所有记录、统计分析、地图可视化轨迹、导出 CSV 报表。

**技术栈：** Flask（后端） + 原生 HTML/CSS/JS（前端） + Leaflet（地图） + JSON 文件存储

---

```
    └── *.csv               ← 每个文件名格式：{日期}_{时间}_{姓名}_{学号}_{ID}.csv
```

## 三、后端：`app.py`（核心文件）

> 所有后端逻辑集中在这一个文件中，共 448 行。

### 3.1 全局配置（第 13-34 行）

| 常量 | 当前值 | 说明 | 修改指引 |
|---|---|---|---|
| `REQUIRE_DURATION_HOURS` | `5` | 要求的最长连续录制时长（小时） | 要改审核标准，改这个数字 |
| `GAP_THRESHOLD_MINUTES` | `10` | 超过此间隔视为录制中断（分钟） | 要改中断判定标准，改这个数字 |
| `UPLOAD_DIR` | `./uploads` | CSV 文件保存路径 | 一般不需要改 |
| `RECORDS_FILE` | `./submissions.json` | 提交记录存储文件 | 一般不需要改 |

### 3.2 工具函数

| 函数 | 位置 | 功能 | 备注 |
|---|---|---|---|
| `load_records()` | 第 21 行 | 从 JSON 读取所有提交记录 | — |
| `save_records(records)` | 第 28 行 | 保存提交记录到 JSON | — |
| `wgs84_to_gcj02(lng, lat)` | 第 72 行 | WGS-84 → GCJ-02 坐标转换 | 当前轨迹 API 未使用 |
| `analyze_csv(csv_path)` | 第 87 行 | **核心审核函数** — 解析 CSV，划分连续时段，计算最长连续录制 | ⭐ 修改审核逻辑改这里 |

### 3.3 API 路由一览

| 路由 | 方法 | 功能 | 对应前端页面 |
|---|---|---|---|
| `/` | GET | 返回首页 | index.html |
| `/submit` | GET | 返回学生提交页 | submit.html |
| `/admin` | GET | 返回管理后台 | admin.html |
| `/api/submit` | POST | **提交作业** — 接收姓名、学号、CSV，保存并自动审核 | submit.html |
| `/api/records` | GET | 获取所有提交记录（简单列表） | submit.html |
| `/api/admin/summary` | GET | 统计总览（总数、通过、未通过、通过率、人数） | admin.html |
| `/api/admin/records` | GET | 分页筛选记录（支持 status + keyword 参数） | admin.html |
| `/api/admin/detail/<id>` | GET | 单条提交详情（重新分析 CSV） | admin.html 弹窗 |
| `/api/admin/export` | GET | 导出 CSV（支持 status 筛选） | admin.html |
| `/api/admin/track/<id>` | GET | 轨迹数据（经纬度 + 时间点） | admin.html 地图 |

### 3.4 常见修改场景

## 四、前端页面

### 4.1 `static/index.html` — 首页（233 行）

**功能：** 系统入口，提供「学生提交」和「管理员入口」两个角色选择卡片。

**结构：**
```
<body>
  .sidebar          ← 左侧导航栏（纯装饰）
  .content-area     ← 右侧主区域
    .topbar         ← 顶栏标题
    .main-scroll    ← 两张角色选择卡片
      .role-card.student  → 跳转 /submit
      .role-card.admin    → 跳转 /admin
```

**特点：** 纯静态，无 JS 交互逻辑。全部 CSS 和 HTML 写在一个文件里。

**常见修改：**
- 修改卡片文案 → 搜索 `学生提交` / `管理员入口`
- 修改卡片颜色 → `.role-card.student::before` / `.role-card.admin::before`

## 五、数据存储

### 5.1 `submissions.json` — 提交记录数据库

JSON 数组格式，每条记录结构：

```json
{
  "submission_id": "B7B639E8",        // 8位提交编号
  "name": "赵宇晨",                    // 学生姓名
  "student_id": "1",                  // 学号
  "filename": "20260426_122528_...",  // 对应的 CSV 文件名
  "submitted_at": "2026-04-26T...",   // 提交时间（ISO格式）
  "status": "审核通过",               // 审核状态
  "max_duration_str": "5小时23分钟",   // 最长连续录制（未通过时有）
  "max_segment_start": "...",         // 最长时段开始时间
  "max_segment_end": "..."            // 最长时段结束时间
}
```

> ⚠️ 已加入 `.gitignore`，不会被提交到 Git。

### 5.2 `uploads/` — CSV 上传文件

文件命名规则：`{YYYYMMDD}_{HHMMSS}_{姓名}_{学号}_{8位ID}.csv`

CSV 内需包含的关键字段：
- `geoTime` — GPS 时间戳（毫秒）
- `latitude` / `lat` / `纬度` — 纬度（任选一种列名）
- `longitude` / `lng` / `lon` / `经度` — 经度（任选一种列名）

## 六、部署与运行

### 本地运行
```bash
cd submission_site
pip install -r requirements.txt
python app.py
# 访问 http://localhost:5000/
```

### 云部署（Render）
- 已配置 `Procfile`：`web: gunicorn app:app`
- 环境变量 `PORT` 会自动设置
- `UPLOAD_DIR` 和 `RECORDS_FILE` 可通过环境变量覆盖
